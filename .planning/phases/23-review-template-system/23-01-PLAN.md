    Create app/core/template_registry.py containing:

    1. TEMPLATE_JSON_SCHEMA dict -- JSON Schema for validating template YAML files. Required fields: name (string minLength 1), version (string pattern digits.digits), source_system (string minLength 1), templates (object). Template entries have optional "desktop" and "mobile" sub-objects. Desktop properties: decision_panel (string), media_player (string), show_scores (boolean), show_candidates (boolean), candidate_layout (string enum ["grid", "side-by-side", "stacked"]). Mobile properties: card_variant (string), show_scores (boolean). Include an "_allowed_partials" list property (array of strings) for whitelisting allowed partial paths.

    2. TemplateConfig dataclass with fields: desktop_decision_panel (str, default "partials/_decision_panel.html"), desktop_media_player (str, default "partials/_media_player.html"), mobile_card_variant (str, default "default"), show_scores (bool, default False), show_candidates (bool, default True), candidate_layout (str, default "grid"), extra_fields (dict, default_factory=dict).

    3. TemplateRegistry class mirroring PolicyEngine structure:
       - __init__ initializes self._templates as dict[str, dict] (source_system -> templates dict)
       - validate_template(yaml_content: str) -> dict: parse with yaml.safe_load, validate against TEMPLATE_JSON_SCHEMA, validate that all include paths in decision_panel/media_player fields start with "partials/" (security check). Raise TemplateValidationError on failure.
       - load_from_file(filepath: str) -> dict: read file, validate, store in registry indexed by source_system.
       - load_from_directory(dirpath: str) -> list[str]: sorted glob of *.yaml, call load_from_file for each.
       - resolve(source_system: str, phase: str | None = None) -> TemplateConfig: lookup source_system in registry, then phase within source templates. Fallback chain: exact phase -> source_system._default -> global default TemplateConfig().
       - list_templates() -> list[str]: return sorted list of loaded source_system names.

    4. derive_source_system(shot_card) -> str standalone function:
       - First check shot_card.narrative_context.get("source_system") (if present and non-empty)
       - Then check shot_card.project_id prefix: starts with "movie-agent" -> "kais-movie-agent", starts with "gold-team" -> "kais-gold-team"
       - Fallback: return "unknown"

    5. Module-level singleton: _registry = None, get_template_registry() -> TemplateRegistry

    Create tests/test_template_registry.py with pytest unit tests covering all 11 behaviors listed above. Use temporary YAML files via tmp_path fixture for loading tests. Test both valid and invalid YAML scenarios.
