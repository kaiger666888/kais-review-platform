**Step 3: Create app/static/sw.js**
Vanilla Service Worker for offline caching:
- On `install`: create cache named `'shot-cards-v1'`, add `/mobile` page shell to cache
- On `activate`: claim clients immediately, delete old caches not matching `'shot-cards-v1'`
- On `fetch`:
  - For requests to `/api/v1/mobile/cards`: network-first strategy with cache fallback. On successful response, cache the JSON response. Store up to 20 Shot Card bundles.
  - For requests to `/mobile` (page navigation): cache-first strategy (serve cached HTML if offline)
  - For other requests: pass through (no caching)
- Use a simple LRU-like approach: when caching cards, if cached card entries exceed 20, delete the oldest
- Implementation detail: store each card response keyed by URL. The list endpoint `/api/v1/mobile/cards?cursor=...` responses are cached individually. Individual card data is also stored in a `shot-card-data` cache as a map of `card-{id}` -> JSON string.

**Alpine.js state management (mobileState function):**
```javascript
function mobileState() {
  return {
    cards: [],           // Array of MobileShotCardBundle
    currentIndex: 0,     // Current visible card index
    candidateIndex: 0,   // Current candidate within card
    cursor: null,        // Pagination cursor
    loading: false,
    hasMore: true,
    showDetail: false,   // Detail panel visibility
    zoomScale: 1,        // Pinch-to-zoom scale
    lastTap: 0,          // Double-tap detection

    // Swipe gesture state
    startX: 0,
    startY: 0,
    deltaX: 0,
    deltaY: 0,
    isSwiping: false,

    // Touch handlers for gesture recognition
    onTouchStart(event) { ... },
    onTouchMove(event) { ... },
    onTouchEnd(event) { ... },

    // Gesture classifiers
    isSwipeLeft() { return this.deltaX < -80 && Math.abs(this.deltaY) < 100; },   // APPROVE
    isSwipeRight() { return this.deltaX > 80 && Math.abs(this.deltaY) < 100; },   // REJECT
    isSwipeUp() { return this.deltaY < -80 && Math.abs(this.deltaX) < 100; },     // SHOW DETAIL
    isSwipeDown() { return this.deltaY > 80 && Math.abs(this.deltaX) < 100; },    // HIDE DETAIL / NEXT

    // Vertical card navigation (shot-to-shot)
    nextCard() { ... },
    prevCard() { ... },

    // Horizontal candidate switching
    nextCandidate() { ... },
    prevCandidate() { ... },

    // Decision actions (call mobile API)
    async approve() { ... },   // POST /api/v1/mobile/cards/{id}/swipe-decision {action:"approve"}
    async reject() { ... },    // POST /api/v1/mobile/cards/{id}/swipe-decision {action:"reject", reason:"Mobile reject"}

    // Pinch-to-zoom
    onPinchStart(event) { ... },
    onPinchMove(event) { ... },
    onPinchEnd(event) { ... },

    // Data loading
    async loadCards() { ... },  // fetch /api/v1/mobile/cards
    async loadAudio() { ... },  // fetch /api/v1/mobile/cards/{id}/audio

    // Initialize
    init() { this.loadCards(); }
  };
}
```

**Gesture implementation details:**
- Use `@touchstart`, `@touchmove`, `@touchend` on the card container div (Alpine.js, NOT external libraries)
- Threshold: 80px minimum for swipe detection (horizontal or vertical)
- Visual feedback during swipe: translate the card in the swipe direction using `:style="'transform: translate(' + deltaX + 'px,' + deltaY + 'px)'"`
- Color overlay during swipe: green tint for left (approve), red tint for right (reject)
- On swipe end: if threshold met, trigger action + animate card off-screen, then show next card
- On swipe end: if threshold NOT met, animate card back to center (spring animation via CSS transition)
- Pinch-to-zoom: detect two-finger touch, track distance changes, apply CSS transform scale
- Swipe down when detail is showing: hide detail (not next card)
- Swipe down when detail is hidden AND at last card: load more cards

**Layout structure:**
```
<body class="bg-gray-900 h-screen overflow-hidden">
  <!-- Context bar (fixed top) -->
  {% include "partials/_mobile_context_bar.html" %}

  <!-- Card viewport (full remaining height, centered) -->
  <div x-data="mobileState()"
       class="relative flex-1 overflow-hidden"
       @touchstart="onTouchStart($event)"
       @touchmove.prevent="onTouchMove($event)"
       @touchend="onTouchEnd($event)">

    <!-- Previous card (peek behind, slightly visible) -->
    <!-- Current card (full size, interactive) -->
    <!-- Next card (peek ahead, slightly visible) -->

    <!-- Use template x-if to render current card -->
    <template x-if="cards.length > 0">
      <div class="absolute inset-x-4 top-4 bottom-4 transition-transform duration-300"
           :style="'transform: translate(' + deltaX + 'px,' + deltaY + 'px) scale(' + zoomScale + ')'">

        <!-- Card content rendered from cards[currentIndex] -->
        {% include "partials/_mobile_card.html" %}

        <!-- Swipe feedback overlays -->
        <div x-show="isSwiping && isSwipeLeft()"
             class="absolute inset-0 bg-green-500/20 flex items-center justify-center rounded-2xl">
          <span class="text-green-400 text-2xl font-bold">APPROVE</span>
        </div>
        <div x-show="isSwiping && isSwipeRight()"
             class="absolute inset-0 bg-red-500/20 flex items-center justify-center rounded-2xl">
          <span class="text-red-400 text-2xl font-bold">REJECT</span>
        </div>
      </div>
    </template>

    <!-- Empty state -->
    <template x-if="cards.length === 0 && !loading">
      <div class="flex items-center justify-center h-full text-gray-500">
        No cards to review
      </div>
    </template>
  </div>

  <!-- Loading indicator -->
  <div x-show="loading" class="absolute bottom-8 inset-x-0 text-center text-gray-400 text-sm">
    Loading...
  </div>
</body>
```

The card partial accesses Alpine.js data via `x-data` scope: `cards[currentIndex]` for the current card fields. Use `x-text` and `:src` bindings to render card data dynamically.
