    4. `async def callback_handler(update: Update, context: ContextTypes.DEFAULT_TYPE) -> None`:
       - Extract callback_query from update
       - Answer the callback query immediately (removes loading spinner): `await callback_query.answer()`
       - Check allowed chat IDs from effective_message.chat.id
       - Parse callback_data: expected format "approve:{review_id}:{version}" or "reject:{review_id}:{version}". If malformed, log error and return.
       - Open DB session via `async_session_factory()`
       - Fetch Review by review_id
       - If review not found, edit message text to "审核不存在" and return
       - If review.state is not APPROVING:
         - If review.state is COMPLETE: edit message with `build_status_text(review, "already_processed", "")` and return (idempotent per CONTEXT)
         - Otherwise: edit message with `build_status_text(review, "stale", "")` and return
       - Determine action: approve -> transition to COMPLETE with disposition HUMAN, reject -> COMPLETE with disposition BLOCK
       - Call `transition_state(session, review_id, ReviewState.APPROVING, ReviewState.COMPLETE, version, actor=f"telegram:{callback_query.from_user.username or callback_query.from_user.id}", action="approve" or "reject", payload={"telegram_user": callback_query.from_user.username, "chat_id": callback_query.message.chat.id})`
       - On success: edit message text with `build_status_text(review, action, actor_name)` using `callback_query.edit_message_text(text=status_text)`
       - On StateConflictError (version mismatch): fetch current review, edit message with current status (idempotent)
       - On other exceptions: log error, edit message with "操作失败，请重试"
