    - Test: routing a ShotCard with no routing_decision raises ValueError
    - Test: enqueue stores shot_id in Redis sorted set with score based on priority + timestamp
    - Test: dequeue removes from Redis sorted set and returns ShotCard data

  <action>
    1. Create `app/core/routing_types.py` with Pydantic models and enums:
       - `Outlet` enum: DESKTOP, MOBILE, AI_AUDIT
       - `Priority` enum: HIGH, LOW
       - `RoutingResult` dataclass: outlet, priority, queued (bool), shot_id
       - `BatchResult` dataclass: approved (list[str]), failed (list[str])
       - `ShotCardRoutedEvent` Pydantic model: shot_id, project_id, outlet, priority, timestamp

    2. Create `app/services/approval_router.py` with `ApprovalRouter` class:
       - Constructor takes redis (aioredis.Redis) and optional event_manager
       - `route(shot_card: ShotCard) -> RoutingResult`: reads shot_card.routing_decision, determines outlet and priority, calls enqueue for HUMAN/AI_AUDIT, directly updates DB for AUTO/BLOCK
       - `_determine_priority(shot_card: ShotCard) -> Priority`: inspect visual_bundle -- if video_clip present -> HIGH, if only keyframes -> LOW. Default HIGH for AI_AUDIT outlet.
       - `_determine_outlet(routing_decision: str) -> Outlet`: HUMAN->DESKTOP, AI_AUDIT->AI_AUDIT
       - `enqueue(shot_id: str, outlet: Outlet, priority: Priority) -> None`: Redis ZADD to sorted set key `queue:{outlet.value}:{priority.value}`, score = priority_weight (HIGH=0, LOW=1000000) + time.time() for FIFO within priority
       - `dequeue(outlet: Outlet, limit: int = 10) -> list[str]`: Redis ZPOPMIN from `queue:{outlet.value}:high` first, then `queue:{outlet.value}:low`. Returns shot_ids.
       - `_auto_approve(shot_card: ShotCard) -> None`: Update ShotCard.audit_status to APPROVED in DB, create AuditEntry
       - `_auto_reject(shot_card: ShotCard) -> None`: Update ShotCard.audit_status to REJECTED in DB, create AuditEntry
       - Emit ShotCardRoutedEvent via event_manager.broadcast() after routing
