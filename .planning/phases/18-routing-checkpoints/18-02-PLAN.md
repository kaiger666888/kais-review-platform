<objective>
Build the Checkpoint Manager for serializing pipeline execution state into Redis, producing ResumeCommands after approval, and implementing the graded timeout policy (24h human auto-reject, 5min AI escalate to human).

Purpose: When a ShotCard enters review, the pipeline is paused. The checkpoint captures all execution state so that after approval, the pipeline can resume from exactly where it stopped. Timeouts prevent cards from languishing indefinitely in review queues.

Output: CheckpointManager service, ResumeCommand model, and arq cron for timeout management.
</objective>

       - `on_approval(shot_id: str, actor: str) -> ResumeCommand | None`: Convenience method: load snapshot -> create resume command -> clear checkpoint. Called by router after approval.
       - `on_rejection(shot_id: str, actor: str, reason: str | None = None) -> None`: Clear checkpoint, emit ShotCardRejectedEvent.
