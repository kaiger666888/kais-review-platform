**Script structure (top to bottom):**

1. **Constants and counters:**
   - `BASE_URL="${BASE_URL:-http://localhost:8090}"` (env var override for non-localhost deployments)
   - `API_KEY="${API_KEY:-CHANGE-ME-USE-openssl-rand-hex-32}"` (matches .env.production default)
   - `API_CONTAINER="review-api"`, `NGINX_CONTAINER="review-nginx"`, `REDIS_CONTAINER="review-redis"` (from docker-compose.yml container_name fields)
   - `PASS=0`, `FAIL=0` counters
   - `pass()` and `fail()` helper functions that print labeled output and increment counters

2. **Helper function `get_auth_token()`:**
   - POST to `$BASE_URL/api/v1/auth/token` with JSON body `{"api_key":"$API_KEY","client_id":"docker-test"}`
   - Extract `.data.access_token` via jq
   - Exit script with error if token is empty/null (auth is prerequisite for most tests)

3. **Pre-flight checks (function `preflight_check()`):**
   - Verify Docker daemon is running: `docker info > /dev/null 2>&1`
   - Verify all 3 containers (review-api, review-nginx, review-redis) are running: `docker ps --format '{{.Names}}'`
   - Wait for API health: poll `$BASE_URL/health` up to 30 times with 2-second sleep, print "Stack is healthy" on success, exit 1 on timeout
   - Print "Pre-flight checks passed" on success

4. **Test DOCK-01 (function `test_dock01`):**
   - `curl -s -o /dev/null -w "%{http_code}" "$BASE_URL/health"` -- note: Nginx routes `/health` not `/api/v1/health`
   - Assert HTTP status is 200
   - Print PASS/FAIL with requirement label "DOCK-01"

5. **Test DOCK-02 (function `test_dock02`):**
   - Parse health response with jq: `curl -s "$BASE_URL/health"`
   - Assert `.status == "ok"` AND `.redis == "ok"` AND `.database == "ok"`
   - Also verify response contains all three keys (status, redis, database)
   - Print PASS/FAIL with requirement label "DOCK-02"

6. **Test DOCK-03 (function `test_dock03`):**
   - Get auth token via `get_auth_token`
   - Submit a review: POST to `$BASE_URL/api/v1/reviews/` with body `{"type":"video","content_ref":"docker-test://test-content","source_system":"docker-test","priority":"normal","risk_score":0.3}` and Bearer auth
   - Extract review_id from `.data.review_id`
   - Generate review token: POST to `$BASE_URL/api/v1/reviews/$REVIEW_ID/token` with Bearer auth
   - Extract token from `.data.token`
   - Cross-verify in Redis: `docker exec review-redis redis-cli GET "review_token:$TOKEN"`
   - Assert Redis value equals the review_id (as string)
   - Print PASS/FAIL with requirement label "DOCK-03"

7. **Main execution:**
   - Call preflight_check, then each test function in sequence (test_dock01, test_dock02, test_dock03 -- test_dock04-07 will be added in Task 2)
   - Print summary: "Results: X passed, Y failed, Z total"
   - Exit 0 if FAIL==0, exit 1 otherwise

**Test DOCK-05 (function `test_dock05`): Total container memory under 400MB:**
