  <action>
    **1. Update Review model in app/models/schema.py:**

    Add two new nullable columns to the Review class, right after `disposition` and before `version`:
    ```python
    callback_url: Mapped[str | None] = mapped_column(String, nullable=True)
    callback_secret: Mapped[str | None] = mapped_column(String, nullable=True)
    ```

    Both columns are `nullable=True` so existing reviews without callbacks work without migration. The `create_tables()` function will handle new installs automatically since it calls `Base.metadata.create_all()`.

    **2. Create RFC1918 validator in a new module app/core/validation.py:**

    Create a standalone function `validate_callback_url(url: str | None) -> str | None`:
    - If `url` is None, return None (callback is optional).
    - Parse the URL using `urllib.parse.urlparse`.
    - Extract hostname, resolve it via `socket.getaddrinfo` (for hostname -> IP resolution).
    - Check that the resolved IP is in RFC1918 private ranges:
      - 10.0.0.0/8 (10.0.0.0 - 10.255.255.255)
      - 172.16.0.0/12 (172.16.0.0 - 172.31.255.255)
      - 192.168.0.0/16 (192.168.0.0 - 192.168.255.255)
    - Also allow 127.0.0.0/8 (loopback) and 169.254.0.0/16 (link-local) for local development.
    - If the IP is NOT in any of these ranges, raise `ValueError(f"Callback URL must resolve to a private IP address, got {ip}")`.
    - Return the original URL if validation passes.

    Use `ipaddress.ip_address` and `ipaddress.ip_network` for the range checks rather than string manipulation.

    **3. Update Pydantic schemas in app/models/schemas.py:**

    In `ReviewCreateRequest`, add after `risk_score`:

    ```

    In `ReviewResponse`, add after `disposition`:

    ```
    Do NOT add `callback_secret` to ReviewResponse -- it must never be exposed in API responses.

    SQLite ALTER TABLE ADD COLUMN with nullable columns works without data loss. Both columns default to NULL for existing rows.
