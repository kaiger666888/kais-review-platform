    1. **`GET /api/v1/audit/stats`** -- Audit statistics aggregation:
       - Query params: `start_date` (ISO, default 7 days ago), `end_date` (ISO, default today), `project_id` (optional filter).
       - Compute from AuditEntry table:
         - `total_decisions`: count of entries where action in ("approve", "reject")
         - `approved`: count where action = "approve"
         - `rejected`: count where action = "reject"
         - `approval_rate`: approved / total_decisions (0 if no decisions)
         - `avg_decision_time_minutes`: average time between first audit entry and approve/reject for each review_id (computed via subquery: for each review_id, find min(created_at) and max(created_at) where action in ("approve","reject"), diff in minutes).
         - `rejection_reasons`: from payload JSON where action="reject", extract payload.reason, group and count top 10. Return as list of {"reason": str, "count": int}.
         - `policy_hit_rates`: from payload JSON where action="reject", extract payload.policy_name (if exists), group and count. Return as list of {"policy": str, "count": int}.
         - `daily_throughput`: for each day in range, count decisions. Return as list of {"date": str, "count": int}.
       - Protect with `require_auditor` dependency (from Plan 02).
       - Return `ApiResponse(data={...all stats...})`.

    2. **`GET /api/v1/audit/timeline`** -- Chronological audit timeline:
       - Query params: `cursor` (AuditEntry id), `limit` (default 50, max 200), `action` (optional filter), `actor` (optional filter), `start_date`, `end_date`.
       - Same query logic as existing `list_audit` but optimized for timeline display:
         - Eagerly join with Review table to include review metadata (type, source_system) in response.
       - Return paginated list with `next_cursor` and `has_more`.
       - Protect with `require_auditor` dependency.
       - Reuse `_audit_response` helper, extend with review metadata.

    3. **`GET /api/v1/audit/policy-diff`** -- Policy version diff:
       - Query params: `commit_sha_1` (required), `commit_sha_2` (required).
       - Use `GitPolicyProvider.get_policies_at_commit(sha)` to load both versions.
       - Compare YAML content of matching policy files between the two commits.
       - Return `ApiResponse(data={"commit_1": sha1, "commit_2": sha2, "diffs": [{"file": str, "from": str, "to": str, "changed": bool}]})`.
       - Protect with `require_auditor` dependency.
       - If git_repo_url is empty, return 503 with error "Git integration not configured".

    Implementation note: Use SQLAlchemy `func.count`, `func.extract`, `case` for aggregation queries. Follow existing pagination pattern from `list_audit`.
