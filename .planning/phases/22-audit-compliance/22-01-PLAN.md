  <action>
    Create `app/core/merkle.py` with:

    1. **MerkleTree class** using Python hashlib:
       - `__init__(self, hashes: list[str])`: Accept ordered list of leaf hashes.
       - `build(self) -> str`: Build binary Merkle tree from leaves. If odd number of leaves, duplicate the last one. Each non-leaf = SHA-256(left_child + right_child). Return the root hash as hex string.
       - `get_leaves(self) -> list[str]`: Return the original leaf hashes.
       - Static method `verify(leaves: list[str], expected_root: str) -> bool`: Rebuild tree from leaves and compare root.

    2. **`compute_daily_merkle_root(target_date: date) -> dict`** async function:
       - Query all AuditEntry rows where `created_at` falls on `target_date` (UTC midnight to midnight), ordered by `created_at ASC`.
       - For each entry, compute leaf hash = SHA-256(f"{entry.id}:{entry.own_hash}") hex string.
       - Build MerkleTree from leaf hashes.
       - Return `{"date": target_date.isoformat(), "root": merkle_root, "leaf_count": N, "leaves": leaf_hashes}`.

    3. **`commit_merkle_root_to_git(merkle_data: dict) -> str | None`** async function:
       - If `settings.git_repo_url` is empty, log warning and return None (graceful degradation).
       - Use gitpython to open/clone the governance repo (same pattern as GitPolicyProvider, using `.policy_repo` local path).
       - Write merkle_data as JSON to `audit_merkle/merkle_{date}.json` in the repo.
       - `repo.index.add([path])`, `repo.index.commit(f"audit: merkle root for {date}")`.
       - Return the commit SHA.

    4. **Verification endpoint** in `app/api/v1/audit_api.py` (append to existing file):
       - `GET /api/v1/audit/merkle/verify?date=YYYY-MM-DD` (default: yesterday).
       - Call `compute_daily_merkle_root(target_date)` to recompute tree.
       - Attempt to read stored `audit_merkle/merkle_{date}.json` from Git repo.
       - If stored root matches recomputed root: return `{"status": "valid", "date": ..., "leaf_count": N}`.
       - If mismatch: return `{"status": "tampered", "date": ..., "stored_root": ..., "computed_root": ...}` with HTTP 409.
       - If no stored root found: return `{"status": "no_anchor", "date": ...}` with HTTP 404.
       - Protect with `require_jwt` dependency (existing auth pattern).

    Edge cases:
       - Empty day (no audit entries): return root = SHA-256("") = "e3b0c44...", leaf_count = 0.
       - Git repo unavailable: log error, return 503 with `{"status": "git_unavailable"}`.
  </action>

    Create `app/core/dual_write.py` with:

    1. **`DualWriteAuditRecorder` class**:
       - `__init__(self)`: No args, reads settings via `get_settings()`.
       - `async def archive_entries_to_minio(self, entries: list) -> int`:
         - For each AuditEntry, serialize as JSON line (fields: id, review_id, action, actor, from_state, to_state, payload, prev_hash, own_hash, created_at ISO format).
         - Collect all lines, write as a single object to MinIO at `audit-archive/{year}/{month}/{day}/entries_{first_id}_{last_id}.jsonl`.
         - Use `_get_minio_client()` pattern from media.py; create bucket if not exists (use `minio_client.bucket_exists` + `make_bucket`).
         - Return count of archived entries.
       - `async def query_warm_storage(self, start_date: date, end_date: date) -> list[dict]`:
         - List objects in MinIO under `audit-archive/{year}/{month}/{day}/` within date range.
         - Read each JSONL object, parse lines into dicts, return flattened list.
         - This enables querying archived data from both hot and warm tiers.

    Create `app/workers/lifecycle.py` with:

    2. **`archive_hot_to_warm(ctx: dict) -> dict`** arq cron task:
       - Called daily (cron at 02:00 UTC).
       - Query AuditEntry rows where `created_at < now() - {hot_retention_days}d` (from settings).
       - Batch in chunks of 500 to avoid memory spikes.
       - For each batch, call `DualWriteAuditRecorder.archive_entries_to_minio()`.
       - After successful MinIO write, delete archived rows from PostgreSQL (using `session.execute(delete(AuditEntry).where(AuditEntry.id.in_(batch_ids)))`).
       - Log archived count via structlog.
       - Return `{"archived": total_count, "batches": batch_count}`.

    3. **`compute_merkle_root_cron(ctx: dict) -> dict`** arq cron task:
       - Called daily (cron at 00:30 UTC, after midnight so all entries for the day are in).
       - Target date = yesterday (UTC).
       - Call `compute_daily_merkle_root(target_date)` from merkle.py.
       - Call `commit_merkle_root_to_git(merkle_data)` from merkle.py.
       - Log result via structlog.
       - Return `{"date": target_date, "root": root_hash, "git_commit_sha": sha}`.

    Register in `app/workers/tasks.py`:
    - Import `archive_hot_to_warm`, `compute_merkle_root_cron` from `app.workers.lifecycle`.
    - Add both to `WorkerSettings.functions`.
    - Add cron jobs: `cron(compute_merkle_root_cron, minute=30, hour=0)`, `cron(archive_hot_to_warm, minute=0, hour=2)`.

    No changes needed in `app/main.py` since WorkerSettings in tasks.py is the single source of truth for arq configuration.

    Edge cases:
       - MinIO unavailable: log error, skip archival (entries remain in PostgreSQL for retry next run).
       - No entries to archive: log info, return early with count 0.
       - Git unavailable for Merkle: log error, return early (Merkle root computed but not anchored).
  </action>
