  <action>
Enhance the `/health` endpoint in `app/main.py` to check real dependency status (DEPL-07):

Replace the current simple `{"status": "ok"}` health endpoint with a version that:
1. Checks Redis connectivity: try `app.state.redis.ping()` if redis exists, report `redis: "ok"` or `redis: "unavailable"`
2. Checks SQLite connectivity: try a simple `SELECT 1` via the database engine, report `database: "ok"` or `database: "error"`
3. Returns HTTP 200 with JSON like `{"status": "ok", "redis": "ok", "database": "ok"}` when all healthy
4. Returns HTTP 503 with `{"status": "degraded", ...}` when any dependency is down
5. The health check must NOT crash if Redis/DB are unavailable - catch exceptions gracefully
6. Import what is needed from existing modules (engine from database, request app state)

Create `scripts/start.sh` as the container entrypoint:
1. `#!/bin/bash`
2. `set -e`
3. `mkdir -p /app/data` (ensure data directory exists for SQLite bind mount)
4. `chmod 777 /app/data 2>/dev/null || true` (ensure writable if running as non-root with bind mount)
5. `exec "$@"` (pass through to Dockerfile CMD)

Update the Dockerfile (from Plan 01) to use this entrypoint:
- Copy `scripts/start.sh` to `/app/start.sh`
- `RUN chmod +x /app/start.sh`
- `ENTRYPOINT ["/app/start.sh"]`
- Keep existing CMD as-is

Since Dockerfile was created in Plan 01, reference the executor to UPDATE the existing Dockerfile with these additions (entrypoint, script copy). Do NOT recreate the whole file.
