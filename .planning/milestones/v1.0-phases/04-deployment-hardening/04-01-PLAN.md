<task type="auto">
  <name>Task 1: Create Dockerfile and .dockerignore</name>
  <files>Dockerfile, .dockerignore</files>
  <action>
Create a production Dockerfile for the FastAPI application:

1. Base image: `python:3.12-slim` (NOT alpine - compiled deps need glibc)
2. Create non-root user `appuser` with UID 1000
3. WORKDIR `/app`
4. Copy `requirements.txt` first, run `pip install --no-cache-dir -r requirements.txt`
5. Copy `app/` directory and any other needed files
6. Create `/app/data` directory owned by appuser (for SQLite bind mount target)
7. USER appuser
8. EXPOSE 8000
9. CMD: `uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 1`
   - Single worker because SQLite single-writer constraint
   - No `--workers` flag needed for MVP (single process)

Create `.dockerignore` excluding:
- `.venv/`, `.git/`, `.planning/`, `__pycache__/`, `*.pyc`
- `.env`, `data/`, `tests/`, `.pytest_cache/`
- `*.md` files (CLAUDE.md, DEPLOYMENT-FEASIBILITY.md, etc.)
- `.claude/`, `.gitignore`

Per DEPL-06: non-root user in Dockerfile. Per DEPL-02: slim image keeps memory low.
  </action>

<task type="auto">
  <name>Task 2: Create docker-compose.yml, nginx.conf, and .env.production</name>
  <files>docker-compose.yml, nginx/nginx.conf, .env.production</files>
  <action>
Create docker-compose.yml (Compose V2, no `version:` key) with 4 services:

**api service:**
- Build from `.` (Dockerfile in root)
- container_name: `review-api`
- Environment: all vars from .env.production
- Volumes: `./data:/app/data` (bind mount for SQLite, DEPL-04)
- Expose port 8000 internally only
- depends_on redis with `condition: service_healthy`
- Memory limit: 256M, CPU limit: 1.0
- Security (DEPL-06):
  - `read_only: true`
  - `tmpfs: ["/tmp"]`
  - `security_opt: [no-new-privileges:true]`
  - `cap_drop: [ALL]`
- Health check: `wget -q --spider http://localhost:8000/health` (30s interval)
- restart: unless-stopped
- Logging: json-file, max-size 10m, max-file 3

**nginx service:**
- Image: `nginx:alpine`
- container_name: `review-nginx`
- Ports: `8090:80` (per CLAUDE.md: http://192.168.71.140:8090)
- Volumes: `./nginx/nginx.conf:/etc/nginx/nginx.conf:ro`
- depends_on api with `condition: service_healthy`
- Memory limit: 32M, CPU limit: 0.25
- Security: `read_only: true`, tmpfs for `/tmp`, `/var/cache/nginx`, `/var/run`
- `cap_drop: [ALL]`, `cap_add: [CHOWN, NET_BIND_SERVICE]`

**redis service:**
- Image: `redis:7-alpine`
- container_name: `review-redis`
- Command: `redis-server --appendonly yes --maxmemory 48mb --maxmemory-policy allkeys-lru` (DEPL-05)
- Named volume `redis_data:/data` for persistence
- Memory limit: 64M, CPU limit: 0.25
- Health check: `redis-cli ping` (30s interval)
- Security: `cap_drop: [ALL]`, `no-new-privileges:true`
- NOT read_only (Redis writes to /data)

**dozzle service (optional profile):**
- Image: `amir20/dozzle:latest`
- container_name: `review-dozzle`
- profiles: `["monitoring"]` (optional, not started by default)
- Volumes: `/var/run/docker.sock:/var/run/docker.sock:ro`
- Ports: `127.0.0.1:9999:8080` (localhost only)
- Memory limit: 32M

**Top-level volumes:** `redis_data: {}` (named volume for Redis AOF)

Create `nginx/nginx.conf` with:
- `worker_processes auto;`
- `events { worker_connections 1024; }`
- `http` block with:
  - Rate limiting zone: `limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;`
  - Rate limiting zone: `limit_req_zone $binary_remote_addr zone=auth:10m rate=5r/s;`
  - `proxy_headers_hash_max_size 512;`
  - Upstream `api { server api:8000; }`
  - `server` block listening on 80:
    - General API: `location /api/` -> proxy_pass to api, rate limit `api` zone, burst 20
    - SSE endpoint: `location /events/stream` -> proxy_pass, NO rate limit, `proxy_buffering off`, `proxy_cache off`, `proxy_http_version 1.1`, `proxy_set_header Connection ''`, `proxy_read_timeout 86400s`, `chunked_transfer_encoding off` (DEPL-03)
    - Web routes: `location /` -> proxy_pass to api (HTMX routes)
    - Health passthrough: `location /health` -> proxy_pass to api
    - Common proxy headers: X-Real-IP, X-Forwarded-For, X-Forwarded-Proto, Host

Create `.env.production` with production-ready defaults:
- `API_KEY=CHANGE-ME-USE-openssl-rand-hex-32`
- `JWT_SECRET=CHANGE-ME-USE-openssl-rand-hex-32`
- `REDIS_URL=redis://redis:6379/0`
- `DATABASE_URL=sqlite+aiosqlite:///./data/review.db`
- `LOG_LEVEL=INFO`

Also update `app/core/config.py` to add a `host` and `port` field (default `0.0.0.0` and `8000`) if not already present, but DO NOT change existing fields.
