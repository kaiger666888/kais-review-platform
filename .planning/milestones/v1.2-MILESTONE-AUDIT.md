## Executive Summary

All 5 phases passed individual verification (VERIFICATION.md: passed, 47/47 observable truths). The review-platform internals are solid — state machine, policy engine, Telegram bot, callback task, and audit log are correctly wired.

**Critical gaps exist at the protocol boundary between review-platform and external systems.** Phase verifiers verified each system in isolation. The integration checker found 4 broken cross-system contracts that make the actual external system integration non-functional:

1. **Auth token field mismatch** (gold-team → review-platform): `review_check.py` reads `data["token"]` but API returns `data["access_token"]`
2. **Callback payload missing field** (review-platform → movie-agent): sends `disposition` but receiver expects `disposition_action`
3. **HMAC header mismatch** (review-platform → gold-team): sends `X-Callback-Signature` but receiver reads `X-Hub-Signature-256`
4. **No CLI entry point** (movie-agent `pipeline.js`): callback-server spawns resume but pipeline.js has no `main()` function

E2E tests masked these by patching `emit_state_change` to a noop, preventing callback delivery from executing during tests.

## Cross-Phase Integration Gaps

### 1. Auth Token Field Name (Critical)

**Files:**
- Broken: `kais-gold-team/kais-hub/worker_node/review_check.py:68,186` reads `data["token"]`
- Correct: `app/api/v1/auth.py` returns `data["access_token"]`
- Also correct: `app/integrations/gold_team/client.py:169` reads `data["access_token"]`

**Impact:** All gold-team worker authentication attempts crash with KeyError. No reviews can be submitted from gold-team.

**Fix:** Change `review_check.py:68` and `:186` from `data["token"]` to `data["access_token"]`

### 2. Callback Payload Contract (Critical)

**Files:**
- Sender: `app/workers/tasks.py:307-312` sends `disposition` (value: "HUMAN")
- Receiver: `kais-movie-agent/bin/callback-server.js:109` expects `disposition_action` (value: "approve"/"reject")

**Impact:** Movie-agent callback-server always gets `disposition_action=undefined`, falls to "Unknown" branch. No resume or rollback ever happens.

**Fix:** Either (a) add `disposition_action` to callback payload in tasks.py based on the action that completed the review, or (b) change callback-server.js to derive approve/reject from the existing `disposition` field or another payload field.

### 3. HMAC Header Name (Medium)

**Files:**
- Sender: `app/workers/tasks.py:325` sends `X-Callback-Signature`
- Receiver: `kais-gold-team/kais-hub/control_node/api/__init__.py:121` reads `X-Hub-Signature-256`

**Additional issue:** HMAC serialization differs (unsorted keys + standard separators vs sorted keys + minimal separators)

**Impact:** Callback delivery to gold-team fails HMAC verification. Currently mitigated by gold-team using polling model, but the callback path is architecturally intended.

### 4. Pipeline.js CLI Entry Point (Critical for movie-agent)

**Files:**
- Spawner: `kais-movie-agent/bin/callback-server.js:150` runs `node lib/pipeline.js resume <phase> --workdir <dir>`
- Target: `kais-movie-agent/lib/pipeline.js` exports only a `Pipeline` class, no `main()` or `process.argv` handling

**Impact:** Even with correct callback payload, the spawned resume process does nothing and exits immediately.

## Root Cause Analysis

The gaps share a common pattern: **phase verifiers verified each system in isolation and did not test the actual protocol between systems.** The E2E tests in Phase 12 were supposed to catch these but patched the callback delivery mechanism (`emit_state_change` → noop) to simplify test setup, which eliminated the very code path that would have revealed the contract mismatches.

**Recommendation:** Fix the 4 broken contracts, then add E2E tests that exercise the actual callback delivery path (without patching emit_state_change).
