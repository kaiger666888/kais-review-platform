Create tests/integration/test_sse_flows.py with class TestSSEFlows.

**Key challenge:** SSE streaming with httpx.AsyncClient requires `client.get(url, headers=..., follow_redirects=True)` but SSE is a long-lived response. Use httpx's streaming response pattern:

```python
async with client.stream("GET", "/api/v1/events/stream", headers=auth_headers) as response:
    # Read events from response.aiter_lines() or response.aiter_bytes()
```

However, SSE tests with actual HTTP streaming are fragile due to timing. A better approach for testing SSE behavior at the HTTP layer:

**Strategy A -- Direct SSE endpoint testing with event_manager manipulation:**
For most tests, connect via HTTP to verify the endpoint works, then use `event_manager.broadcast()` directly (since state transitions call emit_state_change which calls broadcast) and verify events arrive.

**Strategy B -- Two-client pattern:**
One async task connects to SSE and collects events. Another async task triggers state changes. Use `asyncio.wait_for` with timeouts to avoid hanging tests.

**Tests to implement:**

1. `test_sse_connect_and_receive_event` (SSE-01):
   - Connect to /api/v1/events/stream with Bearer auth headers
   - Verify response status 200 and content-type contains "text/event-stream"
   - Use httpx streaming: `async with client.stream("GET", url, headers=auth_headers) as resp`
   - Verify the connection is established (aiter_lines yields something eventually or just verify 200 status)

2. `test_sse_event_on_state_change` (SSE-02):
   - Use the event_manager singleton directly: get reference to `app.core.events.event_manager`
   - Start an SSE connection in background task (collect events into a list)
   - Submit a review via POST /api/v1/reviews (triggers state transition -> emit_state_change -> broadcast)
   - Verify the SSE connection receives a review_status event with matching review_id and state change
   - Use `asyncio.wait_for(collect_task, timeout=5.0)` to avoid hanging

3. `test_sse_heartbeat` (SSE-03):
   - The heartbeat is a 30s timeout in `_event_generator`. Testing actual 30s wait is too slow.
   - Instead, test the heartbeat logic at the unit level by verifying the timeout path: mock asyncio.wait_for to raise TimeoutError, verify ServerSentEvent(comment="heartbeat") is yielded.
   - OR: Accept that heartbeat testing at integration level requires either time manipulation or a shorter timeout for tests. Since the event_manager and generator are well-tested at unit level, verify the SSE endpoint returns content-type and accepts connections, and note heartbeat is unit-tested.
   - PRACTICAL APPROACH: Test that the event_generator yields heartbeat comments by directly testing `_event_generator` with a mocked request that triggers timeout quickly. Or: patch `asyncio.wait_for` timeout to 0.1s in the test and verify heartbeat comment arrives within 1s.

4. `test_sse_disconnect_cleanup` (SSE-04):
   - Get initial event_manager.connection_count
   - Connect to SSE endpoint
   - Verify connection_count increased by 1
   - Close the response/client
   - Wait briefly (asyncio.sleep(0.1))
   - Verify connection_count returned to initial value

5. `test_sse_multiple_clients` (SSE-05):
   - Connect 3 SSE clients concurrently
   - Verify event_manager.connection_count == 3
   - Trigger a state change (submit review)
   - Verify all 3 clients receive the event (each client's event queue gets the data)
   - Use event_manager broadcast directly for reliable testing: broadcast a test event, then read from each SSE response

6. `test_sse_slow_client_dropped` (SSE-06):
   - Create an SSE connection
   - Get the underlying queue from event_manager._connections
   - Fill the queue to maxsize (100 items) by broadcasting 100+ events
   - Broadcast one more event
   - Verify the slow client's queue was removed from event_manager._connections
   - Connect a second "fast" client, broadcast, verify it receives the event even after slow client is dropped

IMPORTANT: All SSE tests must use `@pytest.mark.asyncio` and have reasonable timeouts (use `asyncio.wait_for` with 5-10s max for each test).

IMPORTANT: SSE connections use Bearer JWT auth (the /api/v1/events/stream endpoint). Use the same auth_headers fixture from conftest.py.

IMPORTANT: For streaming responses, use `client.stream("GET", url, headers=headers)` and consume via `response.aiter_lines()`. The aiter_lines() returns lines from the SSE stream. Parse them as "event: ..." and "data: ..." pairs.

IMPORTANT: When testing event_manager connection counts, be aware the SSE endpoint calls event_manager.create_connection() on connect and remove_connection() in the finally block. The _connections set reflects live HTTP connections.
