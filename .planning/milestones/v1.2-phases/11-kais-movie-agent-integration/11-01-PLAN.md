<objective>
Build the movie-agent side of the review platform integration: a Node.js ReviewPlatformClient, replacement of all pipeline review gates with remote review submission, and a callback server that receives approval/rejection results to trigger pipeline resume or rollback.

Purpose: Movie-agent pipeline currently uses a local browser-based review system (interactive-review.js). This plan replaces it with remote review submission to the review platform, enabling Telegram-based approval. The pipeline pauses (exits) after submission and resumes via callback, transforming from synchronous in-process review to asynchronous callback-driven review.

Output: `lib/review-platform-client.js`, modified `lib/pipeline.js` with `_runRemoteReview()`, `bin/callback-server.js` with HMAC verification and resume/rollback spawning.

Create `../kais-movie-agent/bin/callback-server.js`:

A lightweight `node:http` server (no framework) that receives HMAC-SHA256 signed callbacks from the review platform and spawns pipeline resume or rollback as child processes.

Configuration from environment variables:
- `REVIEW_CALLBACK_SECRET` — HMAC shared secret (must match callback_secret in review submission)
- `CALLBACK_PORT` — default 8766
- `PIPELINE_WORKDIR` — default `process.cwd()`

Server logic:
1. Listen for POST `/callback/review_result` only. Return 404 for all other routes.
2. Read request body, verify HMAC signature from `X-Callback-Signature` header using `node:crypto` `createHmac('sha256', secret).update(body).digest('hex')`. Compare `sha256=${expected}` with received signature.
3. If HMAC fails, return 403 with `{"error":"invalid signature"}`.
4. Parse JSON payload, extract `review_id`, `new_state`, `disposition_action` (approve/reject), and `source_system`.
5. Return 200 `{"ok":true}` immediately (don't block the callback delivery).
6. In the background, call `handleCallback(payload)`:
   - Search for `.pipeline-state.json` files that contain the matching `review_id` in their phases. Use the `workdir` from review metadata to locate the correct project directory (RESEARCH.md Pitfall 2 solution).
   - If `disposition_action === 'approve'` or `new_state === 'COMPLETE'` with approved disposition: spawn `node` with `['lib/pipeline-runner.js', 'resume', phaseId, '--workdir', workdir]` as child process using `execFile`. Use `detached: true, stdio: 'ignore'` and call `child.unref()`.
   - If `disposition_action === 'reject'`: spawn rollback via `node bin/git-stage.js rollback ${workdir} ${previousStage}`. Then update pipeline state file: set phase status to `'rejected'`, set `review_id` to null.
7. Add retry logic for review_id lookup: if state file not found or review_id not present yet (RESEARCH.md Pitfall 4), retry with 1s and 3s delays (max 3 attempts).

Print startup message: `Callback server listening on port ${PORT}`.

Include a shebang `#!/usr/bin/env node` and make the file executable.

- Fail-open on review submission failure (logged warning)

**Add new method `_runRemoteReview(phase, phaseConfig)` to the Pipeline class:**

This method replaces the functionality of `_runReview()` for all review gates. The key architectural change: instead of launching a local HTTP server and waiting for browser submission, it submits to the remote review platform, saves pipeline state with review_id, and returns a sentinel that tells the caller to exit the process.

```javascript
async _runRemoteReview(phase, phaseConfig = {}) {
  const config = this.config.reviewPlatform || {};
  const client = new ReviewPlatformClient({
    baseUrl: config.baseUrl || 'http://192.168.71.140:8090',
    apiKey: config.apiKey || process.env.REVIEW_PLATFORM_API_KEY || '',
    timeout: config.timeout || 10000,
  });

  const reviewConfig = phase.review;
  const candidates = phaseConfig.reviewCandidates || reviewConfig.buildCandidates?.(this.workdir) || [];

  // Collect preview images (max 3, base64 encoded)
  const previewImages = await this._collectPreviewImages(phase, candidates);

  try {
    const result = await client.submitReview({
      type: 'pipeline_phase',
      contentRef: `${this.episode}:${phase.id}`,
      metadata: {
        phase_name: phase.name,
        phase_id: phase.id,
        stage_order: phase.stageOrder,
        episode: this.episode,
        workdir: this.workdir,
        candidate_count: candidates.length,
        preview_images: previewImages,
      },
      callbackUrl: config.callbackUrl || `http://192.168.71.38:${config.callbackPort || 8766}/callback/review_result`,
      callbackSecret: config.callbackSecret || process.env.REVIEW_CALLBACK_SECRET || '',
      riskScore: 0.5, // moderate risk for all phases per RESEARCH.md recommendation
    });

    // Save state with review_id
    const state = await this._loadState();
    state.phases[phase.id] = {
      status: 'awaiting_review',
      review_id: result.reviewId,
      submitted_at: new Date().toISOString(),
      routing: result.routing,
    };
    state.currentPhaseId = phase.id;
    await this._saveState(state);

    return { action: 'awaiting_review', review_id: result.reviewId, routing: result.routing };
  } catch (err) {
    // Fail-open: if review platform unreachable, proceed without review (per CONTEXT.md decision)
    console.warn(`[pipeline] Review submission failed, proceeding without review: ${err.message}`);
    return { action: 'approved', selected: [], rejected: [], scores: {}, feedback: {}, skipped: true, error: err.message };
  }
}
```

**Add `_collectPreviewImages(phase, candidates)` method to the Pipeline class:**

This method reads up to 3 image files from candidates and returns them as base64 strings. For audio phases (voice), it returns an empty array (placeholder deferred per CONTEXT.md).

```javascript
async _collectPreviewImages(phase, candidates) {
  const images = [];
  const maxImages = 3;
  for (const candidate of candidates.slice(0, maxImages)) {
    if (candidate.imagePath) {
      try {
        const { readFile } = await import('node:fs/promises');
        const data = await readFile(candidate.imagePath);
        images.push(data.toString('base64'));
      } catch (e) {
        console.warn(`[pipeline] Preview image read failed: ${candidate.imagePath}`);
      }
    }
  }
  return images;
}
```
