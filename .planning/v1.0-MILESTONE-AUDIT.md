## Critical Integration Gaps

### Gap 1: Gold-team Auth Token Field (FATAL)

**Files:**
- Review platform: `app/api/v1/auth.py:25` returns `data["access_token"]`
- Gold-team: `kais-gold-team/.../review_check.py:68,186` reads `data["token"]`

**Impact:** Gold-team cannot authenticate. `KeyError` crash on every auth attempt.
**Fix:** Change gold-team `review_check.py` to read `data["access_token"]`.

### Gap 2: Gold-team HMAC Header Name (FATAL)

**Files:**
- Review platform: `app/workers/tasks.py:325` sends `X-Callback-Signature`
- Gold-team: `kais-gold-team/.../control_node/api/__init__.py:121` reads `X-Hub-Signature-256`

**Impact:** Gold-team always rejects callbacks (401).
**Fix:** Align header name across both systems.

### Gap 3: HMAC Signature Computation (FATAL)

**Files:**
- Review platform: `json.dumps(payload, default=str)` — unsorted keys
- Gold-team: `json.dumps(payload, sort_keys=True, separators=(",", ":"))` — sorted, compact

**Impact:** Even if header name is fixed, HMAC values won't match.
**Fix:** Both sides must use identical JSON serialization.

### Gap 4: HMAC Key Source (FATAL)

**Files:**
- Review platform: Uses `review.callback_secret` (per-review)
- Gold-team: Uses `config["api"]["hmac_secret"]` (global)

**Impact:** Different keys produce different HMACs.
**Fix:** Gold-team must use the same secret submitted as `callback_secret`.

### Gap 5: Callback Payload Field Names (FATAL)

**Files:**
- Review platform sends: `{ disposition: "HUMAN" | "AUTO" | "BLOCK", state: "COMPLETE", ... }`
- Movie-agent expects: `{ disposition_action: "approve" | "reject", ... }`
- Gold-team expects: `{ disposition: "APPROVED" | "REJECTED", ... }`

**Impact:** Neither external system can determine approval vs rejection from callback.
**Fix:** Add explicit `action: "approve" | "reject"` field to callback payload.

### Gap 6: Gold-team Poller Disposition Values (MEDIUM)

**Files:**
- Review platform: `disposition` stores routing decision (HUMAN, AUTO, BLOCK)
- Gold-team poller: checks for `APPROVED` or `REJECTED`

**Impact:** Telegram-approved reviews have `disposition: "HUMAN"`, poller returns "HUMAN" instead of "APPROVED".
**Fix:** Poller should check audit trail for final approve/reject action.

## Assessment

**What works:** Review platform core is fully functional (266 tests). Telegram Bot integration works. Policy engine routes correctly. Callback delivery FROM review platform works. External system client modules are correctly structured.

**What doesn't work:** Cross-system protocol alignment — external systems can't authenticate, verify HMACs, or parse callback payloads.

**Root cause:** External systems were built with their own callback conventions. Review platform was built with its own. Protocols were never aligned at the contract level.

**Recommended fix scope:** ~50 lines across 3 files in external repos + ~10 lines in review platform callback payload.
